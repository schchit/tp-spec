# Time Protocol (TP) Framework

The Time Protocol (TP) is a four-layer conceptual framework for rethinking time in AI systems. **HJS** is a concrete implementation of the TP/3 layer.

## The Four Layers

| Layer | Name | Description | Status |
|-------|------|-------------|--------|
| TP/0 | Time Definition | Core concepts and structure | **[IETF Draft](https://datatracker.ietf.org/doc/draft-wang-tp-definition/)** |
| TP/1 | Time Perception | Modulating time density (slow/fast) | Coming soon |
| TP/2 | Time Direction | Directing time flow (future → present) | Coming soon |
| TP/3 | Time Copy | Copying time across agents with accountability | **[Implemented by HJS](https://github.com/hjs-protocol/spec)** |
| TP/4 | Time Emergence | New dimensions emerging from existing ones | Coming soon |

## IETF Drafts

| Draft | Description | Status |
|-------|-------------|--------|
| [`draft-wang-tp-definition-00`](https://datatracker.ietf.org/doc/draft-wang-tp-definition/) | TP/0: Time Definition Protocol | Active |
| [`draft-wang-hjs-judgment-event-00`](https://datatracker.ietf.org/doc/draft-wang-hjs-judgment-event/) | HJS: Judgment Event Protocol (TP/3 implementation) | Active |

## HJS: The TP/3 Implementation

The [HJS Protocol](https://github.com/hjs-protocol/spec) implements the TP/3 layer by providing:

- **Judgment** — Recording decisions with time context
- **Delegation** — Transferring authority across time
- **Termination** — Ending responsibility chains
- **Verification** — Validating time-based records
